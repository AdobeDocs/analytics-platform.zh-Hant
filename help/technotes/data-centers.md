# Customer Journey Analytics託管位置

Adobe Customer Journey Analytics由北美、歐洲和APAC的公共雲端服務提供者所建立的企業級資料中心所控管。

布建時，客戶需指定其Adobe Experience Platform資料將位於哪個區域。 從Adobe Experience Platform資料湖擷取到Customer Journey Analytics的資料將會儲存在相同區域。

如需詳細資訊，請參閱Adobe Experience Cloud檔案中的[地區資料彙集](https://experienceleague.adobe.com/zh-hant/docs/core-services/interface/data-collection/rdc)。


## 檢視儲存資料的資料中心

>[!NOTE]
>
>無法將資料從一個資料中心移至另一個資料中心。

若要檢視您的資料儲存於哪個資料中心：

1. 登入 [Adobe Experience Cloud](https://experience.adobe.com)。

1. 從介面右上角的應用程式切換器&#x200B;**[!UICONTROL 應用程式]**&#x200B;中選取![Experience Platform](/help/assets/icons/Apps.svg)。

1. 指派給您資料中心的地區碼會顯示在Experience Platform的右上角部分。

   ![Experience Platform首頁上的資料中心](assets/experience-platform-data-center.png)

1. 請使用下表來瞭解您的地區碼會與哪個地理區域相關聯：

   | Adobe區碼 | 雲端提供者 | 地理區域 |
   |-------------------|-------|-------------------------------------------|
   | VA7 | Azure | 美國（預設） |
   | VA6 | AWS | 美國（依請求） |
   | NLD2 | Azure | 荷蘭，阿姆斯特丹 |
   | CAN2 | Azure | 加拿大中部，多倫多 |
   | AUS5 | Azure | 澳洲、雪梨 |
   | GBR9 | Azure | 英國、倫敦 |
   | IND2 | Azure | 印度 |
   | CHE2 | Azure | 瑞士 |
